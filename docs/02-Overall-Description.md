# 2. Overall Description (ภาพรวมของระบบ)

## 2.1 มุมมองผลิตภัณฑ์

### 2.1.1 บริบทของระบบ

ระบบ InventoryPro เป็น **Web Application** สำหรับจัดการสินค้าคงคลังและการเบิกจ่ายสินค้า ออกแบบมาเพื่อใช้งานภายในองค์กรหรือธุรกิจขนาดเล็กถึงกลาง

```
┌─────────────────────────────────────────────────────────────────┐
│                        InventoryPro System                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐          │
│  │   Admin     │    │   Staff     │    │  Customer   │          │
│  │  Dashboard  │    │  Dashboard  │    │  Dashboard  │          │
│  └──────┬──────┘    └──────┬──────┘    └──────┬──────┘          │
│         │                  │                  │                  │
│         └──────────────────┼──────────────────┘                  │
│                            │                                     │
│                   ┌────────▼────────┐                           │
│                   │  React Frontend │                           │
│                   │   (SPA)         │                           │
│                   └────────┬────────┘                           │
│                            │                                     │
│                   ┌────────▼────────┐                           │
│                   │    Firebase     │                           │
│                   │  (BaaS)         │                           │
│                   ├─────────────────┤                           │
│                   │ • Authentication│                           │
│                   │ • Firestore DB  │                           │
│                   │ • Storage       │                           │
│                   └─────────────────┘                           │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 2.1.2 ความสัมพันธ์กับระบบอื่น

| ระบบภายนอก | ความสัมพันธ์ |
|------------|-------------|
| **Firebase Authentication** | ใช้สำหรับยืนยันตัวตนผู้ใช้ (Email/Password) |
| **Cloud Firestore** | ฐานข้อมูล NoSQL สำหรับเก็บข้อมูลทั้งหมด |
| **Firebase Storage** | เก็บไฟล์รูปภาพสินค้าและสลิปการชำระเงิน |
| **Web Browser** | แสดงผล User Interface |

### 2.2.2 รายละเอียดฟังก์ชันหลัก

| รหัส | ฟังก์ชัน | คำอธิบาย |
|------|----------|----------|
| **F01** | Authentication & Authorization | สมัครสมาชิก/เข้าสู่ระบบ/ออกจากระบบ และควบคุมสิทธิ์ตาม Role |
| **F02** | Product Management | เพิ่ม แก้ไข ลบ สินค้า รองรับ Variants และรูปภาพ |
| **F03** | Inventory Management | ติดตามจำนวน Reserved/Available แจ้งเตือนสต็อกต่ำ และประวัติสต็อก |
| **F04** | Shopping Cart | เพิ่ม/ลบสินค้า ปรับจำนวน แยกตาม Role และไปชำระเงิน |
| **F05** | Order Management | สร้างคำสั่งซื้อ/เบิก ติดตามสถานะ และอัปเดตสถานะโดย Admin |
| **F06** | Payment & Coupons | อัปโหลดสลิป ใช้คูปองส่วนลด และยืนยันคำสั่งซื้อ |
| **F07** | User Management | ดู/แก้ไขโปรไฟล์ ที่อยู่ และจัดการผู้ใช้/Role โดย Admin |
| **F08** | Reports & Dashboard | Dashboard สำหรับ Admin และรายการแจ้งเตือนสต็อกต่ำ |
| **F09** | Settings | ตั้งค่าบัญชีธนาคารสำหรับชำระเงิน และการตั้งค่าอื่น ๆ ที่กำหนดไว้ |
| **F10** | Internationalization (i18n) | รองรับการเปลี่ยนภาษาไทย/อังกฤษ |

## 2.3 ลักษณะผู้ใช้งาน

### 2.3.1 ประเภทผู้ใช้งาน

ระบบมีผู้ใช้งาน 3 ประเภท:

#### 1. Admin (ผู้ดูแลระบบ)

| หัวข้อ | รายละเอียด |
|--------|------------|
| **บทบาท** | ผู้ดูแลระบบที่มีสิทธิ์เข้าถึงทุกฟังก์ชัน |
| **ความรับผิดชอบ** | จัดการสินค้า ผู้ใช้ คำสั่งซื้อ คูปอง และการตั้งค่าระบบ |
| **ทักษะที่ต้องการ** | ความเข้าใจพื้นฐานในการใช้งานคอมพิวเตอร์และเว็บแอปพลิเคชัน |
| **ความถี่ในการใช้งาน** | ทุกวัน |

**สิทธิ์การเข้าถึง:**
- ✅ จัดการสินค้า (CRUD)
- ✅ จัดการผู้ใช้และ Role
- ✅ ดูและจัดการคำสั่งซื้อทั้งหมด
- ✅ จัดการคูปอง
- ✅ ดู Dashboard และรายงาน
- ✅ ตั้งค่าบัญชีธนาคาร
- ✅ ดูประวัติสต็อก

#### 2. Staff (พนักงาน)

| หัวข้อ | รายละเอียด |
|--------|------------|
| **บทบาท** | พนักงานที่สามารถเบิกสินค้าสำหรับใช้งานภายใน |
| **ความรับผิดชอบ** | เบิกสินค้า ติดตามคำสั่งเบิกของตนเอง |
| **ทักษะที่ต้องการ** | ความเข้าใจพื้นฐานในการใช้งานเว็บ |
| **ความถี่ในการใช้งาน** | ตามความต้องการ |

**สิทธิ์การเข้าถึง:**
- ✅ ดูรายการสินค้า
- ✅ เพิ่มสินค้าลงตะกร้า
- ✅ สร้างคำสั่งเบิก (Pickup เท่านั้น)
- ✅ ดูคำสั่งเบิกของตนเอง
- ✅ จัดการโปรไฟล์
- ❌ ไม่สามารถจัดการสินค้า
- ❌ ไม่สามารถดูคำสั่งของผู้อื่น

#### 3. Customer (ลูกค้า)

| หัวข้อ | รายละเอียด |
|--------|------------|
| **บทบาท** | ลูกค้าที่สั่งซื้อสินค้า |
| **ความรับผิดชอบ** | สั่งซื้อสินค้า ชำระเงิน ติดตามคำสั่งซื้อ |
| **ทักษะที่ต้องการ** | ความเข้าใจพื้นฐานในการใช้งานเว็บและการซื้อของออนไลน์ |
| **ความถี่ในการใช้งาน** | ตามความต้องการ |

**สิทธิ์การเข้าถึง:**
- ✅ ดูรายการสินค้า
- ✅ เพิ่มสินค้าลงตะกร้า
- ✅ สร้างคำสั่งซื้อ (Pickup/Shipping)
- ✅ ชำระเงินและอัพโหลดสลิป
- ✅ ใช้คูปองส่วนลด
- ✅ ดูคำสั่งซื้อของตนเอง
- ✅ จัดการโปรไฟล์และที่อยู่
- ❌ ไม่สามารถจัดการสินค้า
- ❌ ไม่สามารถดูคำสั่งของผู้อื่น

## 2.4 ข้อจำกัดทั่วไป

### 2.4.1 ข้อจำกัดด้านเทคนิค

| ข้อจำกัด | รายละเอียด |
|----------|------------|
| **Platform** | Web Browser เท่านั้น (ไม่มี Mobile App) |
| **Browser Support** | Chrome, Firefox, Safari, Edge เวอร์ชันล่าสุด |
| **Internet** | ต้องเชื่อมต่ออินเทอร์เน็ตตลอดเวลา |
| **Database** | ใช้ Firebase Firestore (NoSQL) มีข้อจำกัดด้าน Query |
| **File Storage** | จำกัดขนาดไฟล์ตาม Firebase Storage Plan |
| **Concurrent Users** | ขึ้นอยู่กับ Firebase Plan |

### 2.4.2 ข้อจำกัดด้านธุรกิจ

| ข้อจำกัด | รายละเอียด |
|----------|------------|
| **การชำระเงิน** | รองรับเฉพาะการโอนเงินและอัพโหลดสลิป |
| **การจัดส่ง** | ไม่มีการคำนวณค่าจัดส่งอัตโนมัติ |
| **ภาษา** | รองรับ 2 ภาษา (ไทย, อังกฤษ) |
| **สกุลเงิน** | รองรับเฉพาะบาท (THB) |

### 2.4.3 ข้อจำกัดด้านความปลอดภัย

| ข้อจำกัด | รายละเอียด |
|----------|------------|
| **Authentication** | ใช้ Firebase Authentication |
| **Authorization** | ตรวจสอบ Role ที่ Frontend |
| **Data Encryption** | ใช้ HTTPS และ Firebase Security |

## 2.5 สมมติฐานและการพึ่งพา

### 2.5.1 สมมติฐาน

1. **ผู้ใช้มีอินเทอร์เน็ต** - ระบบต้องการการเชื่อมต่ออินเทอร์เน็ตตลอดเวลา
2. **ผู้ใช้มี Browser ที่รองรับ** - ใช้ Browser เวอร์ชันล่าสุด
3. **ผู้ใช้มีความรู้พื้นฐาน** - สามารถใช้งานเว็บแอปพลิเคชันได้
4. **ข้อมูลสินค้าถูกต้อง** - Admin กรอกข้อมูลสินค้าอย่างถูกต้อง
5. **การชำระเงินถูกต้อง** - Customer อัพโหลดสลิปที่ถูกต้อง

### 2.5.2 การพึ่งพา (Dependencies)

#### External Dependencies

| Dependency | เวอร์ชัน | ความสำคัญ |
|------------|---------|-----------|
| Firebase | 12.6.0 | Critical - Backend ทั้งหมด |
| React | 19.2.0 | Critical - Frontend Framework |
| React Router DOM | 7.9.5 | Critical - Routing |
| i18next | 25.7.1 | High - Internationalization |
| date-fns | 4.1.0 | Medium - Date Utilities |

#### Internal Dependencies

| Module | พึ่งพา | รายละเอียด |
|--------|-------|------------|
| Services | Repositories | ทุก Service ใช้ Firestore Repository |
| Pages | Services | ทุก Page เรียกใช้ Services |
| Auth | Firebase Auth | ระบบ Authentication |
| i18n | i18next | ระบบแปลภาษา |

### 2.5.3 ความเสี่ยง

| ความเสี่ยง | ผลกระทบ | การบรรเทา |
|------------|---------|-----------|
| Firebase Downtime | ระบบใช้งานไม่ได้ | ใช้ Firebase Status Page ติดตาม |
| Quota Exceeded | ระบบช้าหรือหยุดทำงาน | Monitor Usage, Upgrade Plan |
| Security Breach | ข้อมูลรั่วไหล | ใช้ Firebase Security Rules |
