# 1. Introduction (บทนำ)

## 1.1 วัตถุประสงค์ของเอกสาร

เอกสารฉบับนี้เป็น Software Requirements Specification (SRS) สำหรับระบบ **InventoryPro** ซึ่งเป็นระบบจัดการสินค้าคงคลังและการเบิกจ่ายสินค้าแบบครบวงจร โดยมีวัตถุประสงค์ดังนี้:

1. **กำหนดขอบเขตและความต้องการของระบบ** - อธิบายฟังก์ชันการทำงานทั้งหมดที่ระบบต้องมี
2. **เป็นเอกสารอ้างอิงสำหรับทีมพัฒนา** - ใช้เป็นแนวทางในการพัฒนาและทดสอบระบบ
3. **สื่อสารกับผู้มีส่วนได้ส่วนเสีย** - ให้ผู้ใช้งานและผู้บริหารเข้าใจความสามารถของระบบ
4. **เป็นพื้นฐานสำหรับการบำรุงรักษา** - ใช้อ้างอิงในการแก้ไขและพัฒนาต่อยอดในอนาคต

---

## 1.2 ขอบเขตของโปรเจค

### 1.2.1 ชื่อระบบ
**InventoryPro - ระบบจัดการสินค้าคงคลังและการเบิกจ่าย**

### 1.2.2 ขอบเขตการทำงาน

ระบบ InventoryPro ครอบคลุมการทำงานดังนี้:

| หมวดหมู่ | ขอบเขต |
|----------|--------|
| **การจัดการสินค้า** | เพิ่ม แก้ไข ลบ สินค้า รองรับ Variants (ไซส์/สี) |
| **การจัดการสต็อก** | ติดตามจำนวนสินค้าคงคลัง แจ้งเตือนสต็อกต่ำ |
| **การเบิกจ่ายสินค้า** | รองรับการเบิกแบบรับเอง (Pickup) และจัดส่ง (Shipping) |
| **การจัดการคำสั่งซื้อ** | สร้าง ติดตาม อัพเดทสถานะคำสั่งซื้อ |
| **ระบบคูปอง** | สร้างและจัดการคูปองส่วนลด |
| **การจัดการผู้ใช้** | ระบบสมาชิก 3 ระดับ (Admin, Staff, Customer) |
| **รายงานและการแจ้งเตือน** | Dashboard สรุปข้อมูล แจ้งเตือนสต็อกต่ำ |
| **รองรับหลายภาษา** | ไทย และ อังกฤษ |

### 1.2.3 สิ่งที่อยู่นอกขอบเขต

- ระบบชำระเงินออนไลน์อัตโนมัติ (ใช้การอัพโหลดสลิปแทน)
- ระบบจัดส่งสินค้าอัตโนมัติ (ใช้การกรอกเลข Tracking แทน)
- ระบบ POS (Point of Sale)
- Mobile Application (รองรับเฉพาะ Web Application)

## 1.3 คำจำกัดความและคำย่อ

### 1.3.1 คำย่อ

| คำย่อ | ความหมาย |
|-------|----------|
| **SRS** | Software Requirements Specification |
| **UI** | User Interface |
| **UX** | User Experience |
| **API** | Application Programming Interface |
| **CRUD** | Create, Read, Update, Delete |
| **SKU** | Stock Keeping Unit |
| **i18n** | Internationalization |

### 1.3.2 คำจำกัดความ

| คำศัพท์ | คำจำกัดความ |
|---------|-------------|
| **Admin** | ผู้ดูแลระบบที่มีสิทธิ์เข้าถึงทุกฟังก์ชัน |
| **Staff** | พนักงานที่สามารถเบิกสินค้าและดูคำสั่งซื้อของตนเอง |
| **Customer** | ลูกค้าที่สามารถสั่งซื้อสินค้าและติดตามคำสั่งซื้อ |
| **Variant** | รูปแบบย่อยของสินค้า เช่น ไซส์ สี |
| **Reserved** | จำนวนสินค้าที่ถูกจองไว้แต่ยังไม่ตัดสต็อก |
| **Withdrawal** | การเบิกจ่ายสินค้าออกจากคลัง |
| **Pickup** | การรับสินค้าด้วยตนเอง |
| **Shipping** | การจัดส่งสินค้า |
| **Coupon** | คูปองส่วนลด |
| **Low Stock** | สถานะสินค้าที่มีจำนวนน้อยกว่า 20% ของจำนวนเริ่มต้น |

## 1.5 ภาพรวมของเอกสาร

เอกสารฉบับนี้แบ่งออกเป็น 6 ส่วนหลัก:

### บทที่ 1: Introduction (บทนำ)
- วัตถุประสงค์ ขอบเขต คำจำกัดความ และภาพรวมของเอกสาร

### บทที่ 2: Overall Description (ภาพรวมของระบบ)
- มุมมองผลิตภัณฑ์ ฟังก์ชันหลัก ลักษณะผู้ใช้งาน ข้อจำกัด และสมมติฐาน

### บทที่ 3: Specific Requirements (ความต้องการเฉพาะ)
- Functional Requirements และ Non-functional Requirements

### บทที่ 4: System Features (คุณสมบัติของระบบ)
- รายละเอียดแต่ละฟีเจอร์ และ Use Cases

### บทที่ 5: External Interface Requirements
- User Interfaces, Hardware, Software และ Communication Interfaces

### บทที่ 6: Other Requirements
- Database, Internationalization และ Legal Requirements
